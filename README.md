# Gallegos Trailer Sales — gallegossales.com

A fast, SEO-focused static website for Gallegos Trailer Sales (Laredo, TX).
Rebuilt from the old GoDaddy Website Builder site to fix the core SEO problem:
the old catalog lived entirely inside images (no real text), so only the home
page was indexed. This version uses **real HTML text**, **one indexable URL per
product category**, unique meta tags, image `alt` attributes, and structured data.

No frameworks and no runtime dependencies — just static HTML, one CSS file, and a
tiny bit of vanilla JavaScript. It loads fast and can be hosted anywhere.

## Project structure

```
/                         generated HTML pages (index.html, bulk/, liquid/, …)
assets/css/styles.css     single global stylesheet (system fonts, no web-font deps)
assets/js/main.js         mobile nav toggle + contact-form handling
assets/img/               product images (SVG placeholders — replace with photos)
scripts/build.mjs         static-site generator (content + layout live here)
scripts/gen-placeholders.mjs   regenerates the SVG placeholder images
sitemap.xml, robots.txt   generated by the build
```

## Editing content / rebuilding

Page content and the shared header/footer are defined **once** in
`scripts/build.mjs`. Edit there, then regenerate the HTML:

```bash
npm run build      # regenerates all *.html + sitemap.xml + robots.txt
```

To preview locally:

```bash
npm run serve      # serves the site at http://localhost:3000
# or simply: python3 -m http.server 3000
```

> The generated `.html` files **are committed** so the site can be hosted with no
> build step. If you edit the HTML by hand instead of the generator, just don't run
> `npm run build` afterward (it would overwrite your manual edits).

## Logo

`assets/img/logo.svg` is a **self-contained vector** recreation of the Gallegos
Trailers logo: a red badge with the "Gallegos" brush-script wordmark (Kaushan
Script, subset + embedded as a base64 data URI so it renders anywhere with no
external font), the dark "Trailers" ribbon, and the MR mark. It has a transparent
background (no gray borders) and stays crisp at any size. The favicon is generated
from the same source. To tweak proportions/colors, edit `scripts/gen-logo.mjs` and
run `npm run logo`.

> If you have the **original logo artwork** (vector `.ai`/`.eps`/`.svg`, or a
> high-resolution PNG), drop it in `assets/img/` and point the header/footer `src`
> at it in `scripts/build.mjs` for a pixel-perfect match — the current file is a
> faithful recreation, not the original vector.

## Replacing the product images

The images in `assets/img/*.svg` are branded placeholders. Replace them with real,
**compressed** photos (`.jpg` or `.webp`, ~1600px wide, optimized) for the best
look and load speed. Two options:

1. **Keep the filenames** — save your photo over the matching file (e.g. a real
   `steel-pneumatic-dry-bulk-trailer.jpg`) and update the `src`/extension in
   `scripts/build.mjs`, then `npm run build`.
2. Drop new files in `assets/img/` and point the `img` references in
   `scripts/build.mjs` at them.

Recommended filenames + `alt` text (already used in the markup, SEO-friendly):

- `steel-pneumatic-dry-bulk-trailer-1400-cubic-feet` — *Steel pneumatic dry bulk trailer with 1,400 cubic feet capacity*
- `aluminum-pneumatic-dry-bulk-trailer-lightweight-tare` — *Lightweight aluminum pneumatic dry bulk trailer with low tare weight*
- `vacuum-tank-trailer-130-150-bbl-oilfield-water-hauling` — *Vacuum tank trailer, 130 to 150 Bbl, for oilfield and produced water hauling*
- `demolition-end-dump-trailer-hardox-450-scrap-metal` — *Demolition end dump trailer in HARDOX 450 steel for scrap metal and debris*

## Enabling the contact form

The form on `/contact/` posts to a form-delivery service. It works on any static
host — no server needed. By default it points at a **placeholder** endpoint, so
you must configure one:

**Option A — Formspree (recommended, works anywhere):**

1. Create a free form at <https://formspree.io> and copy your form ID.
2. In `scripts/build.mjs`, find `formspree.io/f/YOUR_FORM_ID` and replace
   `YOUR_FORM_ID` with your ID.
3. `npm run build`.

The JavaScript submits via AJAX and shows an inline success message. Until an
endpoint is configured it falls back to a normal form submit. If a submission
fails, the form tells the visitor to call or WhatsApp instead.

**Option B — Netlify Forms (if hosting on Netlify):** add `netlify` and
`name="contact"` attributes to the `<form>` in `scripts/build.mjs` and remove the
`action`; Netlify captures submissions automatically.

## Hosting

Any static host works: Netlify, Cloudflare Pages, GitHub Pages, Vercel, or plain
web hosting. The clean category URLs (`/bulk/`, `/liquid/`, …) come from the folder
structure, so most hosts serve them with no extra config. Just deploy the repo root.

## Domain / DNS

The domain **gallegossales.com** stays the same. Once the new site is live on its
host, only the DNS records need to point to the new hosting.

> **Security note:** that DNS change should be made directly by the owner of the
> GoDaddy account. It should **not** be delegated to a third party or an automated
> assistant. This project does not touch DNS.
